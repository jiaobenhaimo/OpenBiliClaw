# 多层网状记忆系统设计

> OpenBiliClaw 的记忆系统是整个 Agent 的核心，决定了 Agent 对用户的理解深度。

## 设计理念

传统推荐系统用"标签"理解用户，而 OpenBiliClaw 用"记忆"理解用户。就像你了解一个好朋友，不是因为你给他贴了标签，而是因为你记得他说过什么、做过什么、在什么情况下开心或难过。

## 五层记忆架构

### 事件层 (Event Layer)
**"他做了什么"**

最底层的原始数据，记录用户在 B 站的每一个行为：
- 点击、搜索、收藏、评论、点赞、投币
- 对应的 DOM 页面快照和浏览路径
- 时间戳和行为上下文
- Agent 推荐后的用户反应

存储：SQLite（结构化事件日志）

### 偏好层 (Preference Layer)
**"他喜欢什么/不喜欢什么"**

从事件层提取的结构化偏好：
- 兴趣标签（带权重和时间衰减）
- 内容风格偏好（时长、节奏、制作质量）
- 情境模式（工作日/周末、碎片/沉浸）
- 探索开放度

存储：JSON（可读可编辑）

### 觉察层 (Awareness Layer)
**"每天他在发生什么变化"**

Agent 每天的观察和思考：
- 每日观察笔记（"今天他搜索了三次摄影相关内容"）
- 兴趣趋势追踪（"对游戏的兴趣在下降"）
- 情绪状态推测（"最近看放松类内容变多了，可能工作压力大"）
- 阶段性总结

存储：JSON + 向量索引（支持语义检索）

### 洞察层 (Insight Layer)
**"为什么他会这样"**

对用户行为的深层解读：
- 动机分析（"他看探店视频不是为了美食，是享受发现的过程"）
- 心理需求推断（"刷游戏实况是为了获得掌控感和放松"）
- 潜在兴趣假设（"他可能会喜欢旅行vlog，因为也满足发现欲"）
- 行为模式归因

存储：JSON（可审查可修正）

### 灵魂层 (Soul Layer)
**"他是一个什么样的人"**

最高层的人格理解：
- 核心性格特质
- 价值观和审美倾向
- 当前生活阶段和状态
- 深层未被满足的需求

存储：JSON（核心记忆，始终在 Agent 上下文中）

## 四种记忆类型

| 类型 | 说明 | 实现 |
|------|------|------|
| 核心记忆 | 始终加载到 Agent 上下文 | 灵魂层 + 偏好层摘要 |
| 情景记忆 | 具体的交互片段 | 事件层 + 觉察层，SQLite + 向量索引 |
| 语义记忆 | 用户相关的事实知识 | 偏好层 + 洞察层，知识图谱/JSON |
| 工作记忆 | 当前会话上下文 | 运行时内存 |

## 层间交互

记忆层之间不是单向流水线，而是**网状双向关联**：

- **自底向上**：新事件 → 更新偏好 → 触发觉察 → 修正洞察 → 塑造灵魂
- **自顶向下**：灵魂理解 → 指导解读 → 关注变化 → 校准权重
- **跨层直连**：重大行为可直接修正灵魂层；灵魂层可影响事件采集策略

## 自我编辑

Agent 可以自主：
- 决定什么信息提升到更高层
- 合并重复或过时的记忆
- 标记需要验证的假设
- 在反馈后调整各层内容
